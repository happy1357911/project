## 6. 目前主要問題

### 問題 1：Task2 CHL 極少

Task2 `CHL` 只有 2 筆。這會造成：

- train/valid split 很容易不穩
- macro-F1 對少數類別非常敏感
- single seed 結果不可靠

建議必看 per-class recall/F1 與 support，不要只看 macro-F1。

### 問題 2：Task2 rule baseline 很強

Task2 label 與 clinical rule mismatch 為 0。這表示模型很難用「比 rule 更準」作為主要貢獻。

建議把研究主張改成：

- model 是否能在 missing / NR / censored 資訊下提供可用預測
- no-support deployment 是否穩定
- multi-task / reward confidence 是否改善不完整證據下的錯誤

### 問題 3：`artificial_missingness` 不能直接當 formal validation

`results_v74/artificial_missingness/artificial_missingness_summary.csv` 的 `scenario=none` 是全資料 inference，不是 train/valid split 的正式 validation。因此簡報或論文中若使用 0.9835 這類數字，必須標清楚它是 artificial-missingness full-data scenario，不應標成 formal no-support validation。

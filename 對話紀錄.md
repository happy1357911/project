## 2026-04-30 AUC Evaluation Fix
- 已修正 `train_v74.py` 的 `safe_multiclass_auc()`。
- 原問題：舊版會把所有 class 一次丟給 `roc_auc_score()`，當 validation 中某些 class 完全沒有正樣本時，sklearn 會輸出 `UndefinedMetricWarning: Only one class is present in y_true`。
- 新作法：逐一 class 計算 one-vs-rest AUC；若該 class 在 validation 中沒有同時具備正樣本與負樣本，則跳過該 class。

- 此修改只影響評估輸出的 AUC 指標，不影響訓練 loss、optimizer、model weights、資料切分或 best checkpoint 的 macro-F1 選擇。

### Key Conclusions
- 缺值資料有成功進入 baseline，且有被 `forced_pred_label` 判斷一次。
- `forced_accuracy` / `forced_macro_f1` 代表「缺值也硬判」的規則結果，容易高估 rule baseline，不建議作為主要論文比較。
- `covered_only` 只看證據完整資料，分母較小，也不適合單獨拿來跟 neural model 主結果比較。
- `abstain_as_error_accuracy` 最符合使用者定義的 baseline 公式：
  - 有成功進入 baseline 且判斷正確的筆數 / 全部有 label 的資料筆數。
  - 缺值、no BC、NR、censored、tympanogram missing 等證據不足資料保留在分母中，若 baseline 無法正式判讀則算作未成功分類。
- 若要證明「模型比臨床規則更能處理缺值」，可用 `full` no-support 結果對照 baseline 的 `abstain_as_error_accuracy`，並同時報告 coverage / abstain rate。
- 若要證明「meta 結構本身對缺值資料判斷有提升」，不能只和 baseline 比；核心比較應該是同一批資料、同一個 missing subgroup 上的 `full` vs `no_meta`。
- baseline 只能作為臨床規則參考基準；`full - no_meta` 才是 meta 結構效果的主要證據。
- macro-F1 比較要注意 label space：`abstain_as_error_macro_f1` 會引入 `INSUFFICIENT_EVIDENCE` 類別，若要和 neural model 的 macro-F1 嚴格比較，建議另做一致 label-space 的 missing subgroup metric。

### Current Recommended Comparison
| Purpose | Main Comparison | Reason |
|---|---|---|
| 證明模型可處理缺值 | `full` no-support vs baseline `abstain_as_error_accuracy` | baseline 遇到證據不足會保留在分母，較接近真實臨床規則限制 |
| 證明 meta 結構有幫助 | `full` vs `no_meta` on missing subgroup | 這才隔離 meta 結構本身的效果 |
| 證明 reward 有幫助 | `full` vs `no_irl` | 對應 reward / expert consistency 的 ablation |
| 證明 forced rule 可能高估 | baseline `forced_*` vs `abstain_as_error_*` | 顯示缺值硬判與保守判讀的差異 |
